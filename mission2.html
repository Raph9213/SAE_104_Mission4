Mission 2 par MAHMUD HASSAN

Contexte

Exploitation BDD KDou avec une interface low code

Les 4 indicateurs que j'ai choisi sont :

Indicateur 1

Énoncé : Donner l’adresse complète du fournisseur Marie Delamare.

Interprétation : Cette requête permet d’extraire l’adresse d’un fournisseur spécifique. On cherche à obtenir les détails (adresse, ville, pays) pour le fournisseur ayant pour nom de société “Marie Delamare”. Si le fournisseur n’existe pas, aucun résultat ne sera retourné. Cela peut indiquer soit une erreur dans la base de données, soit l’absence du fournisseur dans la liste.

Résultat

Société Adresse Ville Pays
       

Code de SQL

Requête SQL
SELECT Fournisseur.Societe, Fournisseur.Adresse, Fournisseur.Ville, Fournisseur.Pays
FROM Fournisseur
WHERE (Fournisseur.Societe = "Marie Delamare");

Indicateur 2

Énoncé : Donner la fonction du contact pour le fournisseur “Ma Maison”.

Interprétation : Cette requête permet de connaître le poste occupé par le contact du fournisseur. On veut récupérer la fonction exacte de la personne associée à la société “Ma Maison”. Cela est souvent utile pour savoir à qui s’adresser dans un contexte commercial.

Résultat

Société Fonction
Ma Maison Directeur du marketing
   

Code de SQL

Requête SQL
SELECT Fournisseur.Societe, Fournisseur.Fonction
FROM Fournisseur
WHERE (((Fournisseur.Societe) = "Ma Maison"));

Indicateur 3

Énoncé : Lister les produits d’Italie ou d’Allemagne dont le prix unitaire est de 50€ ou moins.

Interprétation : Cette requête permet d’extraire une liste de produits abordables provenant d’Allemagne ou d’Italie. Les critères sont basés sur le pays d’origine du fournisseur et sur un prix unitaire inférieur ou égal à 50€. Cela peut être utile pour une étude de marché ou pour identifier des opportunités commerciales à moindre coût.

Code de SQL

Requête SQL
SELECT Produit.NomProd, Produit.PrixUnit, Fournisseur.Pays
FROM Fournisseur INNER JOIN Produit ON Fournisseur.NoFour = Produit.NoFour
WHERE (((Produit.PrixUnit <= 50) AND (Fournisseur.Pays) = "Italie" OR (Fournisseur.Pays) = "Allemagne"));

Indicateur 4

Énoncé : Lister les catégories de produits vendus par les fournisseurs basés à Singapour.

Interprétation : Cette requête permet d’identifier les différentes catégories de produits proposés par des fournisseurs situés à Singapour. Cela peut être utile pour analyser la diversité de l’offre ou pour cibler des fournisseurs spécialisés dans certains types de produits.

Code de SQL

Requête SQL
SELECT Produit.NomProd, Categorie.NomCateg, Fournisseur.Pays
FROM Fournisseur INNER JOIN (Categorie INNER JOIN Produit ON Categorie.CodeCateg = Produit.CodeCateg) ON Fournisseur.NoFour = Produit.NoFour
WHERE (((Fournisseur.Pays) = "Singapour"));

Conclusion

Ces requêtes SQL montrent comment utiliser les commandes SELECT, FROM, et WHERE, ainsi que les jointures (INNER JOIN), pour récupérer des données de plusieurs tables. Chaque requête permet d’extraire des informations spécifiques en filtrant selon des critères comme le nom de la société, le pays du fournisseur ou le prix des produits. Les jointures relient les données entre les tables, ce qui est essentiel pour faire des analyses détaillées dans une base de données.
